Federated Machine Learning

This is a Federated Machine Learning system that uses a Central Coordinator and has learner nodes which are Mutex/Blocking. The learner nodes will train the model one after one another, with the Central Coordinator broadcasting the most up-to-date model back to the learner nodes for the next iteration. On the MNIST dataset, we are getting accuracy that is within 3% of a consolidated model.

Overview

The following diagram shows how the acid works at high-level:



  1. User provides docker-compose.yml with required parameters
  2. YAML file is parsed and parameters are compared with requirements
  3. For missing parameters wizard is activated and prompts user for the missing parameters
  4. Application starts the deployment process for the selected provider
  5. Returns confirmation of successful deployment or errors that are logged to the logs.txt created in the current directory.

Network Compression Detection

Detecting the presence of network compression in cooperative and non-cooperative networks.
For a cooperative network, the presence of compression is detected by comparing the time taken for low-entropy and high-entropy data to be transmitted. A UDP packet train, bookended by TCP packets is used.
For a non-cooperative network, the presence of compression is detected by comparing the time taken for an RST packet to be transmitted. I implemented the standalone application using Raw Sockets.
